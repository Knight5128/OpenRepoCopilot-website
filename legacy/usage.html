从安装到洞察的端到端旅程

一条主线贯穿 OpenRepo Copilot 的日常使用：装好 App 与 skill，启动本地工作台，把 GitHub 仓库或文档变成可探索的知识图谱。

六步走通全流程

运行方式 本地优先 · 绑定 127.0.0.1
数据源 GitHub 公共仓库 / 文档集合
支持格式 .md / .txt / .pdf / .docx
本地存储 ~/.openrepo-copilot

OpenRepoCopilot Workbench

安装 App 与 skill

下载客户端，或运行一键脚本将全套 skill 装入你的 Agent（Claude Code / Codex）。

启动本地工作台

/openrepo 启动 Workbench，浏览器打开 http://127.0.0.1:5173/，本地管理多个项目。

创建项目

填入 GitHub 公共仓库地址，或导入 .md / .txt / .pdf / .docx 文档知识库。

队列分析

/openrepo-analyze <project-id> 领取队列任务，多 Agent 自动执行（github_repo / document_kb 两种流程）。

生成知识图谱

输出 knowledge-graph.json，包含文件、模块、函数、依赖等节点与关联边、分层与导览。

仪表盘探索

/understand-dashboard 可视化探索，并配合 /understand-chat、/understand-diff、/understand-domain、/understand-onboard 深入理解。

一次完整分析长这样

输入一个仓库地址，系统自动创建项目、执行多 Agent 分析、生成交互式知识图谱，全程本地完成。
